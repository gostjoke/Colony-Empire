# 《殖民天下》遊戲設計文件 (GDD)

> 類型：殖民經營 / 4X 策略（城鎮建造 + 科技 + 貿易 + 外交戰爭）
> 時代：1500–1811
> 參考座標：《Sid Meier's Colonization》＋《Anno 1800》＋《Civilization》的勝利條件
> 視角：單機，玩家 vs 多個 AI 國家 + 中立美洲部落 NPC

---

## 1. 設計總綱

### 1.1 核心體驗（玩家應該感受到什麼）
玩家是一支從歐洲出發的殖民隊伍，在新大陸從一艘船、幾個移民開始，逐步建立並壯大殖民地。核心張力來自三條同時推進的軸線：

- **內政**：滿足殖民地居民不斷升級的需求（食物 → 商品 → 服務），人口才會成長與升級。
- **經濟**：建立生產鏈，把原料加工成商品，賣回歐洲賺錢，再投資科技與擴張。
- **外部**：與其他國家、美洲部落競爭土地與資源，可外交合作也可武力併吞，最終走向「分數勝利」或「獨立勝利」。

### 1.2 一句話設計目標
> 「在 311 年（1500–1811）的時間壓力下，把一個移民據點經營成新大陸最強的殖民政權——或乾脆推翻母國自立。」

### 1.3 核心遊戲循環（Core Loop）
```
探索/取得土地 → 建造生產建築 → 滿足居民需求 → 人口成長與升級
        ↑                                              ↓
   外交/戰爭擴張 ← 投資科技/軍事 ← 累積金錢 ← 貿易（賣往歐洲）
```
每一圈都應在 1–3 分鐘內完成一次正回饋，讓玩家持續有「我又前進了一步」的感覺。

## 2. 時間與回合制度

建議採用 **即時 + 可暫停（real-time with pause）** 或 **回合制**，二擇一。下表給出兩種方案，建議新手開發先做回合制（實作單純、易除錯）。

| 方案 | 優點 | 缺點 | 建議 |
|---|---|---|---|
| 回合制（1 回合 = 1 年）| 邏輯清晰、易實作、易做 AI | 節奏較慢 | ✅ 第一版採用 |
| 即時可暫停 | 沉浸感強、像 Anno | 同步/平衡複雜 | 後期再升級 |

- **時間範圍**：1500 → 1811，共 **311 個年回合**（可壓縮，例如 1 回合 = 2 年 → 約 155 回合，避免遊戲過長）。
- **里程碑事件**：每 25 年觸發一次「時代結算」，刷新市場價格、解鎖新移民潮、AI 重新評估外交。
- **獨立倒數**：玩家宣布獨立後，母國於數回合內派遣鎮壓軍，進入限時戰爭狀態。

## 3. 資源與人口系統

### 3.1 兩種「貨幣」
- **金錢（Gold）**：跨大陸通用，靠貿易與稅收取得，用於建造、僱傭、外交。
- **科技點 / 自由點數**：抽象的研發資源，由特定建築（學校、印刷所）與人口產生。

### 3.2 人口階層（居民升級制）
參考 Anno 的居民升級邏輯——人口分階層，每一階層有不同的「需求清單」，需求被滿足才會成長並升級到下一階。

| 階層 | 名稱 | 主要需求（範例） | 產出 |
|---|---|---|---|
| 1 | 移民 / 農夫 | 食物、木材 | 基礎勞力、稅收（低）|
| 2 | 自由民 / 工匠 | + 布料、酒、教堂服務 | 加工勞力、稅收（中）|
| 3 | 商人 / 市民 | + 咖啡、書籍、市政服務 | 高階勞力、稅收（高）、科技點 |
| 4 | 仕紳 / 名流 | + 奢侈品、大學、劇院 | 最高稅收、政治影響力 |

> 設計重點：**每升一階，需求項目變多、變難**，迫使玩家延伸生產鏈與貿易。這是整個遊戲的成長引擎。

### 3.3 需求滿足度 → 成長/衰退
- 每種需求有「覆蓋率（%）」。覆蓋率高 → 人口成長 + 滿意度上升；覆蓋率低 → 停滯或人口流失。
- 滿意度也影響：稅收效率、暴動風險、以及「獨立支持度」（見第 7 章）。

## 4. 經濟、貨物與生產鏈

### 4.1 貨物分類
- **原料**：木材、礦石、棉花、菸草、甘蔗、咖啡豆、毛皮…
- **加工品**：木板、工具、布料、蘭姆酒、雪茄、咖啡、書籍…
- **服務**（非實體，由建築提供範圍服務）：教堂、酒館、市場、學校、醫院、劇院…

### 4.2 生產鏈範例
```
棉花田 → (棉花) → 紡織廠 → (布料) → 滿足自由民需求 / 出口歐洲
甘蔗園 → (甘蔗) → 蒸餾廠 → (蘭姆酒) → 滿足需求 / 出口（高利潤）
森林  → (原木) → 鋸木廠 → (木板) → 建造材料
鐵礦  → (礦石) → 鐵匠舖 → (工具) → 蓋進階建築的前置
```
> 設計原則：**愈下游的加工品愈值錢，但需要愈長的鏈與愈多人口**。出口高階加工品是主要利潤來源。

### 4.3 與歐洲貿易（核心金流）
- 玩家有一條「歐洲航線」：用船把貨物運回母國販售、把移民/軍隊運來。
- **動態市場價格**：每種貨物在歐洲有浮動價。賣得多 → 價格下跌（供需）。鼓勵玩家分散品項、追逐時代需求。
- **航運成本與風險**：航行需時間，海上可能遇海盜或敵國私掠船（後期內容）。
- **關稅張力**：母國對殖民地課稅，稅率隨時間或事件上升 → 成為玩家走向「獨立」的動機之一。

### 4.4 與美洲部落貿易
- 部落會買賣特定貨物（毛皮、食物、特產），價格與歐洲不同，可套利。
- 良好貿易 → 提升部落好感（見第 6 章）。

## 5. 科技樹

科技分四條線，互相牽動。建議用「解鎖式」科技樹（解一項才能解下一項）。

| 科技線 | 範例節點（早 → 晚） | 解鎖內容 |
|---|---|---|
| 生產 | 輪作 → 水車磨坊 → 工廠化生產 | 提升產量、解鎖加工建築 |
| 民生 | 印刷術 → 公共教育 → 啟蒙思想 | 解鎖高階居民需求、加速科技點、提升獨立支持 |
| 軍事 | 火繩槍 → 燧發槍 → 線列步兵/火砲 | 解鎖兵種、提升戰力 |
| 航海 | 卡拉維爾帆船 → 蓋倫帆船 → 護衛艦 | 提升運量、航速、海戰能力 |

- **科技點來源**：高階人口 + 教育類建築。
- **時代鎖**：部分科技需到達特定年代才開放（呼應 1500–1811 的歷史節奏）。
- **「啟蒙思想」是關鍵節點**：解鎖後大幅提升獨立支持度上限，是通往獨立勝利的門檻科技。

## 6. 外交與美洲部落 NPC

### 6.1 對象
- **其他殖民國家（AI）**：不同國家（如西、英、法、荷），各有特性加成（西=軍事、荷=貿易、法=部落外交、英=人口成長）。
- **美洲部落（中立 NPC）**：散布地圖，擁有土地與特產。

### 6.2 與部落的互動選項
- **合作**：貿易、結盟、傳教/文化交流 → 取得土地使用權、特產、傭兵。
- **對抗/併吞**：軍事征服取得其土地與資源，但會降低與其他部落、甚至歐洲輿論的好感（可加入「聲望」懲罰）。
- **好感度系統**：每個部落對玩家有 -100~+100 好感，影響貿易價格、是否開戰、是否提供援軍。

### 6.3 與 AI 國家的外交
- 狀態：和平 / 貿易協定 / 同盟 / 緊張 / 戰爭。
- 可進行：談判邊界、共享地圖、結盟對抗第三方、宣戰、議和。
- AI 行為傾向由「性格參數」驅動（擴張型 / 貿易型 / 守成型）。

## 7. 戰爭與兩種勝利條件

### 7.1 軍事系統（建議從簡）
- 單位：步兵、騎兵、火砲、戰艦 + 移民可武裝成民兵。
- 戰鬥解算：第一版用簡化的「戰力值對比 + 地形/科技加成 + 隨機」即可，不必做即時戰術。
- 補給：軍隊需金錢維持，過度擴軍會拖垮經濟 → 形成戰略取捨。

### 7.2 勝利條件 A：分數勝利（1811 結算）
遊戲在 1811 年結束，計算總分，最高者勝。建議計分權重：

| 項目 | 範例權重 |
|---|---|
| 人口總數與階層 | 30% |
| 累積財富 + 經濟產值 | 25% |
| 領土與城鎮數量 | 20% |
| 科技進度 | 15% |
| 外交聲望（部落/盟友） | 10% |

### 7.3 勝利條件 B：獨立勝利（提前獲勝）
1. 殖民地需累積足夠 **獨立支持度**（由滿意度、啟蒙科技、高階人口、母國稅率不滿等推動）。
2. 達門檻後可 **宣布獨立**。
3. 母國於數回合後派遣 **鎮壓遠征軍**（強度隨玩家規模與年代調整）。
4. 在限時內 **擊退鎮壓軍** → 立即獲得獨立勝利，遊戲結束。
5. 風險：宣布獨立會中斷歐洲貿易與移民補給，需事先自給自足 → 形成高張力的「賭一把」時刻。

> 兩種勝利形成不同打法：**和平經營派**衝分數，**革命派**搶獨立。這是本作的策略深度核心。

## 8. 介面與玩家回饋（UX）

- **主地圖**：殖民地、資源點、部落、敵國一覽。
- **城鎮面板**：各居民階層的需求覆蓋率（綠/黃/紅燈號），一眼看出缺什麼。
- **貿易面板**：歐洲與部落市場價格、航線管理。
- **科技面板**：科技樹進度。
- **外交面板**：各勢力關係與獨立支持度進度條。
- **警示系統**：缺貨、暴動風險、敵軍逼近、市場崩盤要主動提示。

## 9. 實作優先順序（給你的開發 Roadmap）

按「先讓它能玩，再讓它好玩」的順序。每階段都應是可執行的版本。

### MVP（最小可玩版本）— 先做這個
1. 回合制時間推進（年份 + 結束回合按鈕）。
2. 一座殖民地、1–2 個居民階層、3–4 種貨物的生產鏈。
3. 需求滿足 → 人口成長的閉環。
4. 簡單的歐洲貿易（固定或半動態價格）。
> 目標：玩家能「種田→加工→賣錢→人口長大」，循環跑得通。

### 第二階段 — 讓它有策略
5. 科技樹（先做生產線 + 民生線）。
6. 動態市場價格（供需）。
7. 第三、四居民階層與更長的生產鏈。

### 第三階段 — 加入對手
8. 美洲部落 NPC + 好感度 + 貿易。
9. 1 個 AI 殖民國家（先做最簡單的擴張 AI）。
10. 基礎軍事與征服。

### 第四階段 — 完整勝利系統
11. 外交系統完整化、多個 AI 國家。
12. 分數勝利結算（1811）。
13. 獨立支持度 + 獨立戰爭。
14. 平衡性調整、事件、音效美術。

## 10. 技術建議（依你會寫部分 coding）

| 需求 | 建議選項 | 理由 |
|---|---|---|
| 引擎 | **Godot**（GDScript/C#）或 **Unity**（C#） | 2D 城鎮建造皆勝任；Godot 免費輕量、適合獨立開發 |
| 純網頁版 | **TypeScript + Phaser / PixiJS** | 易分享、易迭代 |
| 資料驅動設計 | 把「貨物、建築、科技、需求」全寫成 **JSON/CSV 設定表** | 平衡調整不用改程式，這對這類經營遊戲極重要 |
| 架構 | 把「遊戲狀態」與「渲染」分離（MVC / ECS） | 回合邏輯易測試、AI 易接入 |

> 強烈建議：**第一週只做 MVP 第 1–4 點的純文字/極簡介面原型**（甚至不畫圖，用按鈕和數字），先驗證「生產鏈閉環好不好玩」，再投資美術。
