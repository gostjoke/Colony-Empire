# Colony Empire 開發教學 — 第一課
## 從零開始:安裝 Godot，做出可放置建築的等距網格

歡迎!這一課結束時,你會擁有一張會發光的等距地圖,滑鼠移到哪格亮哪格,左鍵蓋房子、右鍵拆房子。這就是城市建設遊戲最核心的互動骨架。

我已經把第一課「寫好的成品」放在 `godot_starter` 資料夾,你可以先跑起來看效果,再回來一行一行看懂它。**但我強烈建議你跟著下面的步驟自己做一遍**——從零打出來,才學得進去。

---

## Lesson 0：先建立心理模型(2 分鐘)

在動手前,先懂三個詞,後面就不會迷路:

- **引擎 (Godot)**:你的工作室。提供畫面、輸入、物理、編輯器。
- **場景 (Scene, .tscn)**:一個可重複使用的「東西」。一棟房子是一個場景、一個居民是一個場景、整張地圖也是一個場景。場景可以互相嵌套。
- **節點 (Node)**:場景由節點組成。節點是最小積木,例如「Node2D」(2D 空間裡的物件)、「Sprite2D」(圖片)、「Label」(文字)。
- **腳本 (Script, .gd)**:掛在節點上的程式,決定它的行為。我們用 **GDScript**,語法很像 Python。

一句話:**節點組成場景,腳本賦予行為。**

---

## Lesson 1：安裝 Godot 4

1. 打開瀏覽器,到官網 `https://godotengine.org/download` 。
2. 下載 **Godot Engine 4.x**(選「Standard」版,不要 .NET/C# 版,我們先用 GDScript)。
3. Godot 是一個 zip,**解壓縮後直接是一個 .exe,雙擊就能用**,不需要安裝。建議把它放到固定資料夾,例如 `E:\Tools\Godot\`。
4. 雙擊開啟,你會看到「專案管理員」。

> 小提醒:Godot 整個引擎只有約 100 MB,啟動秒開——這正是它適合新手的原因之一。

## Lesson 2：開啟起步專案

我已經幫你建好一個可直接開的專案,先用它確認環境沒問題:

1. 在 Godot 專案管理員,點右上角 **Import(匯入)**。
2. 選到 `E:\Colony Empire\godot_starter\project.godot`,按 **Import & Edit**。

4. 你應該看到一張綠色等距網格。移動滑鼠會高亮格子,**左鍵蓋房子、右鍵拆房子**。

如果這一步成功,代表 Godot 與你的電腦都沒問題。接下來我們「假裝它不存在」,自己從空白做一遍。

## Lesson 3：自己從空白建立專案

1. 專案管理員 → **Create / 新建專案**。
2. 專案名稱填 `colony_learn`,選一個空資料夾,Renderer 選 **Compatibility**(相容模式,對舊顯卡與筆電最穩),按 **Create & Edit**。
3. 進入編輯器,左上「Scene(場景)」面板會說「沒有根節點」。點 **Other Node**,搜尋 `Node2D`,選它。這就是我們地圖的根。
4. 在場景面板把這個節點改名為 `Main`(右鍵 → Rename)。
5. 按 **Ctrl+S** 存檔,檔名 `main.tscn`。
6. 再到「Project → Project Settings → Application → Run」,把 **Main Scene** 設成 `main.tscn`(這樣按播放才知道要跑哪個場景)。

現在你有一個空舞台了。

## Lesson 4：理解等距座標(這是重點)

城市建設遊戲的靈魂,就是「**格子座標**」和「**螢幕像素**」之間的換算。

我們的地圖是一格一格的菱形。每格用 `(col, row)`(第幾欄、第幾列)表示。但畫到螢幕上時,菱形要「斜著排」,所以要換算:

```
格子(col,row) → 螢幕(x,y):
    x = (col - row) × 格寬/2
    y = (col + row) × 格高/2
```

反過來,滑鼠在螢幕上點了 `(x,y)`,要算出它落在哪一格:

```
螢幕(x,y) → 格子:
    col = ( x/(格寬/2) + y/(格高/2) ) / 2
    row = ( y/(格高/2) - x/(格寬/2) ) / 2
```

等距菱形的慣例是**格高 = 格寬的一半**(例如 64×32),看起來才像 2:1 的標準等距視角。

> 你不必背公式,但要理解:「螢幕上斜斜的菱形格,背後其實是一張整齊的方格表」。所有城市建設遊戲都是這個把戲。

## Lesson 5：貼上腳本,讓它動起來

1. 在場景面板點選 `Main` 節點。
2. 點面板上方的「捲軸+綠色」**Attach Script(附加腳本)** 圖示。
3. 路徑用預設 `res://main.gd`,按 Create。
4. Godot 會打開程式編輯器,把裡面預設內容**全部刪掉**,改貼上 `godot_starter/main.gd` 的內容(那個檔案我已經幫你寫好並加滿中文註解)。
5. 按 **F5 播放**。成功的話,你會看到和起步專案一樣的可互動地圖。

接著,打開 `main.gd`,對照下面的「程式地圖」逐段讀懂它:

- **常數區(const)**:網格大小、每格尺寸。改 `GRID_W`、`GRID_H` 看地圖變大變小。
- **`grid_to_screen` / `screen_to_grid`**:就是 Lesson 4 的兩條公式。
- **`_process`**:Godot 每一幀自動呼叫。我們用它持續偵測「滑鼠現在在哪一格」。
- **`_unhandled_input`**:處理滑鼠點擊,左鍵把該格記進 `buildings`,右鍵移除。
- **`_draw`**:真正畫圖。先畫所有地面菱形,再畫有建築的格子。
- **`queue_redraw()`**:告訴 Godot「畫面變了,請重畫」。Godot 不會每幀重畫,要你主動要求。

---

## 動手實驗(最重要的學習)

讀懂之後,試著改改看——改壞了再改回來,是最快的學法:

1. 把 `GRID_W` 和 `GRID_H` 改成 `20`,地圖變多大?
2. 把草地顏色 `Color(0.30, 0.55, 0.30)` 改成你喜歡的顏色(三個數字是紅、綠、藍,範圍 0~1)。
3. 把房子高度 `h` 從 `30.0` 改成 `60.0`,房子會變高樓。
4. 進階:讓左鍵不能蓋在「已經有房子」的格子上(提示:在 `_unhandled_input` 裡加一個 `if buildings.has(cell): return`)。

## 你現在學到的觀念

- Godot 的節點 / 場景 / 腳本架構。
- 等距座標與螢幕座標的雙向換算(城市建設遊戲的地基)。
- `_process`(每幀更新)、`_input`(處理輸入)、`_draw`(繪圖)三大生命週期。
